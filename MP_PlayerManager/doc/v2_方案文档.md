## 1. 核心定位

**目标**：在 FreeLoadout Mod 框架内扩展，新增**角色模板配置 + 存档管理**功能，服务于多人联机开黑场景。

**使用流程**：
1. 玩家在游戏内打开 FreeLoadout 面板（F1）
2. 配置角色模板（选择角色 + 卡牌 + 遗物 + 药水）
3. 导出模板为 JSON 文件
4. 游戏关闭后，用外部工具读取模板 + 存档，向 `current_run_mp.save` 注入新玩家

**v2 核心优势**（相对于 v1）：

| 维度 | v1 | v2 |
|------|:---:|:---:|
| 数据来源 | extracted/ 静态数据 | 游戏内存实时数据（ModelDb） |
| UI 风格 | 标准 PyQt5 | 复用 FreeLoadout 风格（Game feel） |
| 资产打包 | 大量图标打包进 exe | 零资产，纯引用游戏内置资源 |
| 体积 | ~8.4MB | ~数百 KB |
| 操作时机 | 仅游戏关闭后 | 游戏内即可配置模板 |

### 3.2 UI 框架核心

**主面板**：`LoadoutPanel.cs` — `static class LoadoutPanel`

```
Build()
 ├── CanvasLayer (Layer=100) — 覆盖整个视口
 │    ├── ColorRect 半透明遮罩（点击关闭）
 │    └── PanelContainer（深色背景，圆角）
 │         └── VBoxContainer（主布局）
 │              ├── HBoxContainer — Tab 按钮行
 │              │    ├── Button[7] — tab.cards / relics / potions / powers / events / encounters / character
 │              │    ├── HBoxContainer — contextBar（上下文工具栏）
 │              │    └── Button[✕] — 关闭
 │              ├── ColorRect — 分隔线
 │              └── ScrollContainer
 │                   └── VBoxContainer — contentContainer（Tab 内容）
 └── CanvasLayer (Layer=101) — HoverTip 层
```

**样式规范**（可直接复用）：

| 颜色变量 | 值 | 用途 |
|---------|-----|------|
| `StsColors.cream` | `Color(0.91f,0.86f,0.75f)` | 正文文字 |
| `StsColors.gold` | `Color(0.85f,0.73f,0.35f)` | 标题/高亮 |
| `StsColors.gray` | `Color(0.5f,0.5f,0.5f)` | 禁用/空状态 |
| `StsColors.red` | `Color(0.7f,0.2f,0.15f)` | 删除/危险 |
| `StsColors.blue` | `Color(0.4f,0.6f,1f)` | 特殊高亮 |

**背景色**：`Color(0.08f, 0.06f, 0.1f, 0.95f)`（深紫黑）

## 4. 功能模块设计

### 4.1 Mod 端功能（FreeLoadout 扩展）

#### 4.1.1 新增 Tab：模板配置（TemplatesTab）

在 FreeLoadout 面板中新增一个 **Templates Tab**，结构如下：

```
┌─ Templates ─────────────────────────────────────┐
│  [导出模板] [导入模板] [模板库管理]               │
│  ─────────────────────────────────────────────  │
│                                                  │
│  【角色选择】                                    │
│  [铁甲] [静默] [故障] [亡灵] [储君] [+ 更多角色]  │
│                                                  │
│  ─────────────────────────────────────────────  │
│  【卡组配置】  [浏览全部卡牌] [当前卡组: 5张]      │
│    ▼ 攻击 (3)                                    │
│      [打击×2] [重锤]                             │
│    ▼ 技能 (2)                                    │
│      [防御×2] [盾击]                             │
│  [+ 添加卡牌]  [- 移除选中的]                    │
│                                                  │
│  ─────────────────────────────────────────────  │
│  【遗物配置】  [当前: 燃烧之血]                    │
│    64×64图标网格 [选择遗物]                       │
│                                                  │
│  ─────────────────────────────────────────────  │
│  【药水配置】  [槽1: 空] [槽2: 空] [槽3: 空]      │
│    64×64图标网格 [选择药水]                       │
│                                                  │
│  ─────────────────────────────────────────────  │
│  【模板元信息】                                   │
│    名称: [我的铁甲速攻]                           │
│    备注: [用于开黑配置]                          │
│                                                  │
│         [保存模板]      [导出到文件]              │
└──────────────────────────────────────────────────┘
```

**实现要点**：

- **卡组编辑**：复用 `CardsTab` 的 `CreateNCardWrapper` 方法展示卡牌，扩展 `CardEditPanel` 为选择模式
- **遗物编辑**：复用 `RelicsTab` 的 `AddRelicEntry` 方法，扩展为多选模式（模板允许选 N 件遗物）
- **药水编辑**：复用 `PotionsTab` 的逻辑，支持配置 0-3 瓶药水
- **模板存储**：保存到 `data/user_templates.json`，格式见 §4.2

#### 4.1.2 模板导入/导出

**导出**：将 `user_templates.json` 中的指定模板序列化，保存到用户指定路径（通过 Godot FileDialog）

```csharp
// 导出路径建议
user://MP_PlayerManager/templates/{template_name}.json
```

**导入**：从外部 JSON 文件读取模板，合并到 `user_templates.json`

#### 4.1.3 Steam 好友列表获取（Mod 端）

```csharp
// 通过 Steamworks .NET API 获取好友列表
// 路径：K:/杀戮尖塔mod制作/STS2_mod/SharedConfig/ 参考 Steamworks.NET 的用法

// 获取好友信息
SteamFriends.GetFriend(steamId).GetPlayerName()  // 昵称
// steamId → 用于关联存档中的 players[].steam_id
```

**输出**：生成 `steam_friends_cache.json`，格式：

```json
{
  "76561198679823594": "煎包",
  "76561199032167696": "小明"
}
```

此文件由**外部工具**读取，用于存档界面显示友好名称。

#### 4.1.4 Mod 端需 Hook 的位置

| 功能 | Hook 位置 | 说明 |
|------|-----------|------|
| 注册 Templates Tab | `LoadoutPanel.Build()` → 在 tabs 数组追加 | 无需修改现有逻辑 |
| 模板持久化 | Mod 初始化时加载 `user_templates.json` | Mod卸载/重装后保留模板 |
| Steam好友获取 | `TrainerBootstrap.Init()` | 启动时异步获取 |

### 4.3 外部工具（继承 v1 manage_players.py）

外部工具在 v1 基础上做以下修改：

| 改动 | 说明 |
|------|------|
| **新增：模板导入** | 从 `user_templates.json` 或外部 JSON 读取模板 |

## 7. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **Mod 语言** | C#（与 FreeLoadout 一致） | 复用现有框架 |
| **UI 框架** | FreeLoadout 原生 Godot API（Mod）；PyQt-SiliconUI（工具） | 无需引入额外依赖 |
| **模板存储** | JSON 文件（user_templates.json） | 简单、可移植 |
| **Steam 好友** | Steam Web API（HTTP 请求） | 无需 Steam SDK，直接读取缓存 |
| **外部工具** | **Python + PyQt-SiliconUI** | 灵动优雅，详细见 `tools/PYQT_SILICON_UI_ANALYSIS.md` |
| **存档处理** | Python 标准库 json | .save 文件是纯 JSON，直接读写 |
| **构建工具** | dotnet CLI（C#）；PyInstaller（Python exe） | 标准构建流程 |

## 8. 已排除的功能

以下 v1 时代的功能在本版本中**明确排除**：

| 排除项 | 原因 |
|--------|------|
| 工具内置卡牌/遗物图标 | 复用游戏内置资源，零体积 |
| 模板预览（伤害估算） | 不做预览，直接选 |
| 独立 PyQt 风格 UI | 改用 **PyQt-SiliconUI**（灵动优雅） |
| P2P Steam 连接 | 仅用 Steam 读好友列表 |
| 游戏内热修改存档 | 存档修改在游戏关闭后进行 |
