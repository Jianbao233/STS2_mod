## 1. 背景与目标

为 Slay the Spire 2 多人联机场景制作一个**离线存档修改工具**，用于：
1. **夺舍玩家**：接管离线玩家的状态继续游戏
2. **添加新玩家**：复制现有玩家或以初始状态加入
3. **移除离线玩家**：清理离线玩家的所有数据

---

## 2. 方案总览（v1 时代三选一）

| 方案 | 描述 | 选择 |
|------|------|------|
| **方案A** | 制作配套 Mod（游戏内运行，自动导出数据） | ❌ 未采用 |
| **方案B** | 纯工具方案（用 GodotPCKExplorer 提取 PCK + DLL 反编译生成静态数据） | ❌ 未采用 |
| **方案C** | 混合方案（静态数据 + Steam好友直连） | ❌ 未采用 |

**最终选择：方案A 变体（纯离线工具 + extracted/ 静态数据）**

理由：v1 定位为**独立 exe**，不依赖 Mod，数据来源为 `extracted/` 目录。

---

## 3. v1 架构

```
独立 exe（Python + PyInstaller）
├── steam_friends.py   ← Steam Web API 获取好友昵称
├── game_data.py      ← 从 extracted/ 读取静态数据
├── save_io.py        ← JSON 存档读写 + 备份
├── template_engine.py← 角色模板 CRUD
└── manage_players.py← 主程序 + PyQt5 UI
```

---

## 4. Steam 好友方案（v1）

```
Steam Web API: https://steamcommunity.com/profiles/{steam64}/friends
GET 请求，返回 JSON 格式好友列表

缓存：24小时，存放在 ~/.mp_player_manager/steam_cache.json
```

## 6. v1 局限性（v2 改进方向）

| v1 痛点 | v2 改进 |
|---------|---------|
| extracted/ 数据与游戏版本可能不同步 | 使用游戏内存实时数据（ModelDb） |
| 工具体积大（图标打包进 exe） | 复用游戏内置资产，体积小 |
| UI 简陋（标准 PyQt5） | 使用 PyQt-SiliconUI 风格 |
| 无法在游戏内操作 | 集成到 FreeLoadout Mod 内 |
| 无模板预览（5.4节需求） | 不做预览，直接选 |
