# 安卓端开发与调试工作流

> 本文档记录在真机（安卓手机）上进行 STS2 Mod 开发调试的完整流程。

---

## 一、安卓 Mod 存放位置

游戏实际读取的目录为**内部应用私有存储**，位于：

```
/data/data/com.megacrit.sts2/files/mods/
```

该目录通过 `run-as com.megacrit.sts2` 可访问，普通 shell 无法直接访问（无 root 权限时）。

**外部存储路径**（`/storage/emulated/0/Android/data/com.megacrit.sts2/files/mods/`）**游戏不读取**，仅作辅助参考。

## 三、调试工作流（Debug 覆盖）

适用场景：开发过程中快速验证修改，无需打包发布。

### 3.1 前提条件

- ADB 连接正常（`adb devices` 显示设备）
- 游戏未运行（部署前需关闭游戏）
- Mod 已通过 `build.ps1` 完成本地构建

### 3.2 操作步骤

**Step 1：构建 Mod**（在 PC 上）

```powershell
# 进入 Mod 目录，运行构建脚本
cd "K:\杀戮尖塔mod制作\STS2_mod\NoClientCheats"
.\build.ps1
```

构建产物会自动复制到 Steam mods 目录（`K:\SteamLibrary\steamapps\common\Slay the Spire 2\mods\NoClientCheats\`）和 `torelease\` 快照目录。

**Step 2：推送文件到手机临时目录**

```powershell
# 将构建产物通过 adb push 推到手机 /data/local/tmp/
adb push "K:\SteamLibrary\steamapps\common\Slay the Spire 2\mods\NoClientCheats\NoClientCheats.dll" /data/local/tmp/
adb push "K:\SteamLibrary\steamapps\common\Slay the Spire 2\mods\NoClientCheats\NoClientCheats.pck" /data/local/tmp/
adb push "K:\SteamLibrary\steamapps\common\Slay the Spire 2\mods\NoClientCheats\mod_manifest.json" /data/local/tmp/
```

> `/data/local/tmp/` 对所有用户可读可写。

**Step 3：通过 run-as 复制到内部 mods 目录**

由于 `run-as` 环境内 shell 重定向不可用，需使用 `dd` 管道方式：

```bash
# 覆盖 DLL
adb shell "run-as com.megacrit.sts2 dd if=/data/local/tmp/NoClientCheats.dll bs=1 | run-as com.megacrit.sts2 dd of=files/mods/NoClientCheats.dll bs=1"

# 覆盖 PCK
adb shell "run-as com.megacrit.sts2 dd if=/data/local/tmp/NoClientCheats.pck bs=1 | run-as com.megacrit.sts2 dd of=files/mods/NoClientCheats.pck bs=1"

# 覆盖 manifest
adb shell "run-as com.megacrit.sts2 dd if=/data/local/tmp/mod_manifest.json bs=1 | run-as com.megacrit.sts2 dd of=files/mods/mod_manifest.json bs=1"
```

**Step 4：验证**

```bash
adb shell "run-as com.megacrit.sts2 ls -la files/mods/"
```

确认文件大小与 PC 端一致。

**Step 5：启动游戏测试**

在手机上启动游戏，观察 logcat 中 `[NoClientCheats]` 相关日志。

**Step 6：清理临时文件（可选）**

```bash
adb shell "rm /data/local/tmp/NoClientCheats.dll /data/local/tmp/NoClientCheats.pck /data/local/tmp/mod_manifest.json"
```

### 3.3 快捷命令汇总

一条命令搞定推送（PowerShell）：

```powershell
$files = @("NoClientCheats.dll","NoClientCheats.pck","mod_manifest.json")
$src = "K:\SteamLibrary\steamapps\common\Slay the Spire 2\mods\NoClientCheats"
foreach ($f in $files) {
    adb push "$src\$f" /data/local/tmp/
    $base = [Convert]::ToBase64String([IO.File]::ReadAllBytes("$src\$f"))
    adb shell "echo '$base' | base64 -d | run-as com.megacrit.sts2 dd of=files/mods/$f bs=1"
}
```

## 四、正式发布工作流（Release 打包）

适用场景：版本发布，需要生成 zip 安装包，交给用户手动导入。

### 4.1 完整流程

```powershell
# Step 1: 构建
cd "K:\杀戮尖塔mod制作\STS2_mod\NoClientCheats"
dotnet build -c Debug
# 或 .\build.ps1

# Step 2: 导出 PCK（如有 Godot）
# Godot --path . --export-pack "Windows Desktop" NoClientCheats.pck --headless

# Step 3: 打包
cd "K:\SteamLibrary\steamapps\common\Slay the Spire 2\mods\NoClientCheats"
Compress-Archive -Path "NoClientCheats.dll","NoClientCheats.pck","mod_manifest.json" -DestinationPath "NoClientCheats-v{版本}.zip" -Force

# Step 4: 推送到手机供手动导入
adb push "NoClientCheats-v{版本}.zip" /storage/emulated/0/我的文件/
```

### 4.2 手动导入步骤（手机端）

1. 手机打开文件管理器，进入 `/我的文件/`
2. 找到 zip 文件，解压到同目录
3. 将解压出的文件复制到游戏 mods 目录：`/data/data/com.megacrit.sts2/files/mods/`
4. 启动游戏

> 也可通过游戏内置的 Mod Manager 从外部存储导入（视游戏版本支持情况而定）。

## 五、ADB 连接问题排查

### 5.1 设备未找到

```powershell
adb kill-server
adb start-server
adb devices
```

确认手机已开启开发者选项 → USB 调试（或无线调试）。

### 5.2 WiFi 连接

```powershell
# USB 连接状态下
adb tcpip 5555
# 断开 USB，记住手机 IP（如 192.168.1.100）
adb connect 192.168.1.100:5555
```

### 5.3 验证 logcat 可用

```bash
adb shell "logcat -d -t 3 | tail -5"
```

### 5.4 确认 run-as 可用

```bash
adb shell "run-as com.megacrit.sts2 ls files/mods/"
```

如报错 `run-as: package not debuggable`，说明包未设置 `debuggable=true`，但仍可通过 `dd` 管道方式写入（见 3.3 节）。

## 六、日志查看

### 6.1 通过 adb logcat（推荐）

```bash
# 过滤 godot 日志
adb logcat -d | grep godot | tail -50

# 过滤 Mod 相关日志
adb logcat -d | grep -E "NoClientCheats|ModListHider|NCC"

# 实时查看
adb logcat -c && adb logcat | grep -E "godot|NCC"
```

### 6.2 通过 logfox 应用

手机安装 logfox 后，可直接在应用内查看完整日志，搜索关键词定位问题。

### 6.3 关键日志关键字

| 场景 | 关键字 |
|------|--------|
| Mod 加载 | `Loading assembly DLL` |
| Mod 初始化 | `ModuleInit.Initialize()` |
| Mod 加载完成 | `Finished mod initialization` |
| 运行状态 | `-- RUNNING MODDED!` |
| NCC 作弊拦截 | `NoClientCheats` |
| 反射调用 | `LanConnectBridge` |

## 七、注意事项

1. **调试前关闭游戏** — Mod 文件在游戏运行时可能被锁定
2. **每次覆盖前确认版本** — 避免新旧文件混淆
3. **注意 Mod 是否有独立配置文件** — NCC 无单独 `.json`，但其他 mod（如 `UnlimitedMultiplayer.dll`）可能有对应的 `.json` 配置，也需要同步更新
4. **PC Steam 调试 ≠ 手机调试** — 部分行为仅在手机上出现（如反射调用失败），两者测试环境独立
5. **Godot 路径** — 需在 PC 上有 Godot 4.5.1 Mono 版方可导出 PCK，当前构建脚本会自动搜索常见路径
