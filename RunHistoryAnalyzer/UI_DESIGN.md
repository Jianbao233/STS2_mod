# RunHistoryAnalyzer — UI/UX 设计方案

> 本文档描述按需检测模式的交互设计，供确认后执行。

---

## 一、核心交互流程

```
玩家打开百科 → 历史记录
    ↓
玩家点击某条历史记录（选定对局）
    ↓
【分析】按钮出现（该记录对应的面板旁）
    ↓
玩家点击【分析】
    ↓
├─ 检测中：按钮变为进度状态
├─ 检测完成 → 弹出结果窗口
└─ 无异常 → Toast 提示"未发现异常"
    ↓
结果窗口：显示所有异常（高/中/低）
    ↓
如有大量异常 → 【导出报告】按钮
```

---

## 二、【分析】按钮设计

### 2.1 放置位置

参考 NoClientCheats F6 面板的布局：**记录详情面板的底部操作栏**，紧邻现有的"再来一局"等按钮。

### 2.2 按钮状态

| 状态 | 外观 |
|---|---|
| 默认 | 绿色底 + "🔍 分析" |
| 检测中 | 灰色 + "分析中..."（禁用点击） |
| 已分析-无异常 | 绿色 + "✓ 未发现异常" |
| 已分析-有异常 | 橙色 + "⚠ 存在异常 (N)" |
| 分析失败 | 红色 + "✗ 分析失败" |

### 2.3 缓存机制

- 每条历史记录只分析一次（除非文件被外部修改）
- 分析结果缓存在内存 `Dictionary<文件路径, List<Anomaly>>` 中
- 文件修改时间变化 → 清除该条缓存，重新分析

## 三、异常结果窗口设计

### 3.1 窗口基本属性

| 属性 | 值 |
|---|---|
| 标题 | "分析报告 — {角色} {难度}A 难度" |
| 尺寸 | 560 × 420（固定） |
| 模态 | 是（阻止背景交互） |
| 关闭 | 右上角 × 按钮 + ESC 键 |

│  │    预期：523  实际：1000  偏差：477        │  │
│  ├───────────────────────────────────────────┤  │
│  │ 🟧 [中] 卡牌来源追溯                       │  │
│  │    无法追溯来源的卡牌：STRIKE_P(2张)       │  │

### 3.3 异常列表样式规则

| 等级 | 左侧色条 | 图标 | 颜色 |
|---|---|---|---|
| High | 红色 `#E74C3C` | 🔴 | 红色文字标题 |
| Medium | 橙色 `#F39C12` | 🟠 | 橙色文字标题 |
| Low | 绿色 `#2ECC71` | 🟢 | 绿色文字标题 |

每条异常结构：
```

    详细说明文字                       ← 缩进，普通字重
    补充数据（如有）                   ← 缩进，灰色小字
```

## 四、导出报告功能

### 4.1 触发条件

**手动触发**：用户点击【导出报告】按钮时立即导出，不限数量。

### 4.2 导出格式

**纯文本格式（.txt）**，方便玩家直接发送给他人查看：

```
=== RunHistoryAnalyzer 检测报告 ===
分析时间：2026-03-21 15:30:00
对局时间：2026-03-20 22:00:00
角色：铁甲战士 | 难度：普通 | 结果：胜利
种子：XXXXXXXX

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
检测结果：3 项异常
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【高】金币不守恒
  预期金币：523
  实际金币：1000
  偏差：477
  可能原因：直接编辑存档 / gold 控制台作弊 / 内存修改

【中】卡牌来源追溯
  无法追溯来源的卡牌：
    - STRIKE_P × 2
  可能原因：card 控制台作弊 / 存档直接添加

【低】HP边界异常
  战斗中出现 HP 超出最大上限
  最大HP：80 → 实际显示：82

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
本报告由 RunHistoryAnalyzer 自动生成
仅供辅助参考，无法作为严格仲裁依据
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4.3 导出位置

- 调用系统保存对话框，让玩家自行选择保存位置
- 默认文件名：`runhistory_report_{角色}_{时间戳}.txt`

## 五、无异常时的呈现

### 5.1 按钮反馈

分析按钮变为 **"✓ 未发现异常"**（绿色），保持3秒后恢复默认状态。

### 5.2 可选：Toast 提示

在屏幕底部弹出轻量提示（不影响当前操作）：

```
✓ 该局历史记录未检测到异常
```

## 六、按钮与窗口的技术实现

### 6.1 分析按钮

在 `NRunHistory` 的记录详情面板（Record Detail Panel）底部，找到现有的按钮容器（HBox），在其中添加一个 `Button`，文字为"🔍 分析"。

点击后调用检测逻辑，结果返回后：
- 无异常 → 更新按钮文字 + Toast
- 有异常 → 创建并弹出结果窗口

### 6.2 结果窗口

继承 Godot `Window`（或 `PopupPanel`），用 GDScript 或 C# 创建均可。由于本项目已有 C# 基础，建议继续用 C#。

结构：
```
AnalyzeResultWindow (Window)
  ├─ VBoxContainer
  │    ├─ TitleLabel
  │    ├─ SummaryLabel
  │    ├─ ScrollContainer > VBoxContainer (异常列表)
  │    └─ HBoxContainer (操作栏)
  │         ├─ ExportButton
  │         └─ CloseButton
```

### 6.3 文件导出

Godot 的 `FileDialog`（保存模式）：
```csharp
var dialog = new FileDialog();
dialog.FileMode = FileDialog.ModeEnum.SaveFile;
dialog.Access = FileDialog.AccessEnum.Filesystem;
dialog.Filters = new[] { "*.txt ; 文本文件" };
dialog.CurrentPath = 默认路径;
```
